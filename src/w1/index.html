  4. github 가입
    1. github.com
    2. 이하의 사항에 유의하여 가입.
      • username은 아이디와 같은 역할을 한다. 따라서 다른 사람과 같을 수 없다. 다음에 주의하여 username을 만든다. 이는 앞으로 파일이름이나 variable 이름을 만들 때도 마찬가지니 잘 기억하고 습관화하자.
        1. - (하이픈 혹은 대쉬라 부르는) 것으로 시작하거나 끝내지 않는다.
        2. 오직 영문과 숫자만 사용한다.
        3. 당연히 `~!@#$%^&*()=+<>,./?\|[]{}"' 를 포함한 특수문자는 사용하지 않는다.
        4. 사용할 수 있는 특수문자는 - 와 _ 로, 띄어쓰기를 대신해 사용된다.
        5. 그렇다. 띄어쓰기를 사용하면 안된다.
        6. 대문자도, 소문자도 사용할 수 있지만 대문자와 소문자를 구분한다. (app≠apP≠aPp≠App≠ApP≠APp≠APP) 그러니 띄어쓰기 구분을 제외하면 대문자를 사용하지 말자.
        7. 그렇다. 띄어쓰기 구분을 목적으로 대문자를 쓰는건 괜찮다. (applePenPineapplePen)
          • 프로그래밍에서 띄어쓰기 처리 방법들
            Camel Case
            applePenPineapplePen
            JS를 포함한 대부분의 언어에서
            Kebab Case
            apple-pen-pineapple-pen
            HTML과 CSS에서
            Snake Case
            apple_pen_pineapple_pen
            variable이 아닌 constant 이름에. 더 정확히 constant를 표현할 때는 모두 대문자로 쓰는 경우가 많다. APPLE_PEN_PINEAPPLE_PEN
            Pascal Case
            ApplePenPineapplePen
            Class 이름에
        8. (사실 그럴 필요는 없지만) 숫자로 시작하지 않는다.

    • 영상과 아래 내용은 차이가 있다. 영상은 아래 내용에 대한 참조로만 하고, 영상과 다른 모든 사항은 아래 내용을 따르라.
